Your project has already evolved into a clean layered CAD architecture, which is great for long-term growth. Below is a clear overview of how everything fits together.

High-Level Architecture
Pillar.UI (WPF application)
        ↓
Pillar.Rendering (interaction + visualization)
        ↓
Pillar.Core (CAD model + entities)

Each layer has a specific responsibility.

1. UI Layer

Project: Pillar.UI
Framework: WPF

Responsibilities:

Window layout
Viewport control
Mouse / keyboard events
Tool switching

Key components:

MainWindow.xaml
MainWindow.xaml.cs
Viewport3DX

Important connection:

MainWindow
   ↓
SceneManager
ProjectionService
Active Tool

Rendering uses Helix Toolkit SharpDX via Viewport3DX.

2. Rendering / Interaction Layer

Project: Pillar.Rendering

Responsibilities:

Scene rendering
Viewport interaction
Screen → world projection
Temporary preview geometry
Interaction tools

This layer translates user interaction into CAD operations.

SceneManager

Controls what appears in the viewport.

Scene structure:

Viewport3DX
   ├── Grid
   ├── EntityRoot
   │      └── permanent CAD entities
   └── PreviewRoot
          └── temporary geometry

Responsibilities:

Render entities
Track visual ↔ entity mapping
Show preview geometry
Clear previews
Projection System

Files:

ProjectionService
ViewportExtensions
Workplane

Purpose:

mouse screen position
       ↓
camera ray
       ↓
workplane intersection
       ↓
3D world coordinate

This enables CAD interactions like drawing.

Interaction Tools

Folder:

Pillar.Rendering.Tools

Example tool:

LineTool

Responsibilities:

Interpret mouse events
Call projection system
Create preview geometry
Add entities to document

Example workflow:

Mouse click
   ↓
LineTool
   ↓
ProjectionService
   ↓
world coordinates
   ↓
create LineEntity
   ↓
document updated
3. Core CAD Layer

Project: Pillar.Core

Responsibilities:

CAD data model
Entities
Document storage
Geometry definitions

This layer is UI-agnostic.

CadDocument

Central CAD model:

CadDocument
   └── Entities (ObservableCollection)

SceneManager listens to changes:

Entities.CollectionChanged
        ↓
SceneManager.RenderAll()
Entities

Example:

CadEntity (base)
   └── LineEntity

Represents geometric primitives.

Example:

LineEntity
   Start : Vector3
   End   : Vector3
Full Interaction Pipeline

When drawing a line:

Mouse click
   ↓
MainWindow
   ↓
LineTool
   ↓
ProjectionService
   ↓
World coordinate
   ↓
LineEntity created
   ↓
CadDocument.Entities.Add()
   ↓
SceneManager detects change
   ↓
Renderer creates visual
   ↓
Viewport3DX displays line
Rendering Pipeline
LineEntity
   ↓
LineRenderer
   ↓
LineGeometryModel3D
   ↓
HelixToolkit SharpDX
   ↓
DirectX GPU rendering
Current Scene Graph
Viewport3DX
   ├── Lights
   ├── Grid
   ├── EntityRoot
   │      └── Line visuals
   └── PreviewRoot
          └── temporary preview line
