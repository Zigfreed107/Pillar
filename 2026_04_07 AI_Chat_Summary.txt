PROJECT: Graphite CAD Application (C# WPF + HelixToolkit SharpDX)

## OVERVIEW

Building a Windows desktop CAD application using:

* C#
* WPF
* HelixToolkit.Wpf.SharpDX (DirectX 11 renderer)

Goal:
Create a scalable CAD architecture with proper separation between:

* Core (CAD logic)
* Rendering (Helix SharpDX)
* UI (WPF)
* Interaction (Tools)

---

## ARCHITECTURE

Solution Structure:

Pillar.Core

* Entities (LineEntity, etc.)
* CadDocument (ObservableCollection of entities)
* Snapping (SnapPoint, SnapType)
* Future: Commands (Undo/Redo)

Pillar.Rendering

* SceneManager (renders entities into Viewport3DX)
* EntityRenderers (LineRenderer, etc.)
* Tools (LineTool, ITool - interaction layer)
* Math

  * ProjectionService (screen → world)
  * Workplane (plane intersection)
* Snapping

  * SnapManager (cached snap points)

Pillar.UI

* WPF MainWindow
* Viewport3DX (Helix SharpDX)
* Event routing (mouse → tools)

---

## KEY SYSTEMS IMPLEMENTED

1. Viewport Setup (Helix SharpDX)

* Viewport3DX configured with:

  * hx:PerspectiveCamera
  * DirectionalLight
  * ShowViewCube / ShowCoordinateSystem
* Required fix:

  * DataContext = this (for bindings to work)

2. SceneManager

* Maintains mapping:
  Dictionary<Element3D, CadEntity>
* Uses:

  * _entityRoot (permanent geometry)
  * _previewRoot (temporary geometry)
* Automatically updates on document changes

3. Projection System (CRITICAL)

* Converts screen → 3D world coordinates
* Uses:

  * Viewport.UnProject()
  * Ray → Workplane intersection
* Workplane:

  * Origin = (0,0,0)
  * Normal = (0,0,1) → Z-up

4. Z-Up Coordinate System

* Camera UpDirection = (0,0,1)
* ModelUpDirection = (0,0,1)
* XY plane used as drawing plane

5. Line Tool (Interactive)

* Two-click workflow:
  click → start
  click → end
* Uses ProjectionService
* Adds LineEntity to CadDocument

6. Preview Geometry

* Temporary line shown during mouse move
* Implemented via _previewRoot
* Cleared after final click

7. Snap System (Efficient Design)

* SnapPoint model (Position + Type)
* Entities provide snap points (endpoints, midpoint)
* SnapManager:

  * caches snap points
  * rebuilds only when document changes
* Runtime:

  * simple distance check (fast)

## IMPORTANT DESIGN DECISIONS

1. Separation of Concerns

Core:

* No WPF
* No Helix
* Pure CAD logic

Rendering:

* Helix SharpDX
* SceneManager
* Projection

UI:

* WPF only
* forwards input

Interaction:

* Tools interpret input and call core

---

2. Two Tool Layers

Core Tools:

* Operate on CAD data only

Rendering Tools (Interaction Tools):

* Handle mouse input
* Use projection
* Call core logic

---

3. Performance Strategy

Avoid:

* computing snap points every frame
* scanning all geometry per mouse move

Use:

* cached snap points
* lightweight distance checks
* (future) spatial indexing (KD-tree / quadtree)
